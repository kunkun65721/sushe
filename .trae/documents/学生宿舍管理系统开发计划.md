# 学生宿舍管理系统开发计划

## 一、项目概述

基于若依框架二次开发的学生宿舍管理系统（SuShe），采用前后端分离架构。

### 技术栈

- **后端**: Spring Boot 4.0.3 + MyBatis + Druid连接池
- **前端**: Vue + Element UI
- **数据库**: MySQL
- **项目结构**: 多模块Maven项目（sushe-admin, sushe-system, sushe-framework, sushe-common, sushe-generator, sushe-quartz）

***

## 二、业务需求分析

### 核心功能模块

| 模块     | 功能             |
| ------ | -------------- |
| 宿舍楼管理  | 宿舍楼信息增删改查、楼层管理 |
| 宿舍房间管理 | 房间信息管理、床位管理    |
| 学生管理   | 学生信息、住宿分配      |
| 住宿管理   | 入住登记、退宿管理、床位调整 |
| 宿舍卫生检查 | 卫生检查记录、评分管理    |
| 宿舍报修   | 报修申请、报修处理      |
| 访问记录   | 访客登记           |

***

## 三、数据库设计

### 3.1 宿舍楼表 (my\_dormitory\_building)

| 字段                | 类型          | 说明          |
| ----------------- | ----------- | ----------- |
| building\_id      | bigint(20)  | 宿舍楼ID，主键    |
| building\_name    | varchar(50) | 宿舍楼名称       |
| building\_type    | char(1)     | 楼栋类型（0男 1女） |
| floor\_count      | int(4)      | 楼层数         |
| room\_count       | int(4)      | 房间数         |
| create\_build\_id | bigint(20)  | 创建者         |
| create\_time      | datetime    | 创建时间        |

### 3.2 宿舍房间表 (my\_dormitory\_room)

| 字段            | 类型          | 说明                |
| ------------- | ----------- | ----------------- |
| room\_id      | bigint(20)  | 房间ID，主键           |
| building\_id  | bigint(20)  | 所属宿舍楼             |
| room\_number  | varchar(20) | 房间号               |
| floor\_number | int(4)      | 所在楼层              |
| bed\_count    | int(4)      | 床位总数              |
| bed\_occupied | int(4)      | 已占用床位             |
| room\_type    | varchar(20) | 房间类型（4人间、6人间等）    |
| room\_status  | char(1)     | 房间状态（0空置 1满员 2维修） |

### 3.3 学生表 (my\_student)

| 字段              | 类型          | 说明      |
| --------------- | ----------- | ------- |
| student\_id     | bigint(20)  | 学生ID，主键 |
| student\_name   | varchar(30) | 学生姓名    |
| student\_number | varchar(20) | 学号      |
| gender          | char(1)     | 性别      |
| phone           | varchar(11) | 联系电话    |
| major           | varchar(50) | 专业      |
| class\_name     | varchar(50) | 班级      |
| create\_time    | datetime    | 创建时间    |

### 3.4 住宿记录表 (my\_accommodation)

| 字段                | 类型         | 说明           |
| ----------------- | ---------- | ------------ |
| accommodation\_id | bigint(20) | 住宿ID，主键      |
| student\_id       | bigint(20) | 学生ID         |
| room\_id          | bigint(20) | 房间ID         |
| bed\_number       | int(4)     | 床位号          |
| check\_in\_date   | date       | 入住日期         |
| check\_out\_date  | date       | 退宿日期         |
| status            | char(1)    | 状态（0在住 1已退宿） |

### 3.5 卫生检查表 (my\_dormitory\_inspection)

| 字段               | 类型           | 说明      |
| ---------------- | ------------ | ------- |
| inspection\_id   | bigint(20)   | 检查ID，主键 |
| room\_id         | bigint(20)   | 房间ID    |
| inspector        | varchar(50)  | 检查人     |
| inspection\_date | datetime     | 检查日期    |
| score            | int(4)       | 评分      |
| remark           | varchar(500) | 备注      |

### 3.6 报修记录表 (my\_repair)

| 字段           | 类型           | 说明                 |
| ------------ | ------------ | ------------------ |
| repair\_id   | bigint(20)   | 报修ID，主键            |
| room\_id     | bigint(20)   | 房间ID               |
| student\_id  | bigint(20)   | 报修学生               |
| repair\_type | varchar(50)  | 报修类型               |
| description  | varchar(500) | 报修描述               |
| status       | char(1)      | 状态（0待处理 1处理中 2已完成） |
| apply\_date  | datetime     | 申请日期               |
| handle\_date | datetime     | 处理日期               |

## 八、开发规范

1. **代码生成器**: 利用若依自带的代码生成器生成增删改查基础代码
2. **命名规范**: 遵循若依框架现有命名规范
3. **分层结构**: Controller → Service → Mapper → Domain
4. **前端规范**: 复用现有Vue组件和API调用模式
